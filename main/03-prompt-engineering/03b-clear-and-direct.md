# 03b - 清晰且直接

提示词的第一行是整个请求中最重要的部分，它是为后续所有内容设定基调的地方。关键是要清晰和直接，使用简单明了的语言，确保阐述“你希望 Claude 做什么”时没有歧义。

![img](./03b-clear-and-direct.assets/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623557%2F05_-_002_-_Being_Clear_and_Direct_02.1748623557175.png)

清晰指的是：

- 使用简单易懂的语言
- 直接说明你想要什么，不要拐弯抹角
- 明确陈述 Claude 的任务

例如：

```
好的提示词：
    写三段关于太阳能板如何工作的内容
不好的提示词：
    我需要了解那些放在屋顶上利用太阳的东西，就是那些“太阳能板”（我感觉它们叫那个）
```

直接指的是：

- 写指令，而不是写问题
- 以行动动词开头，比如：写、创建、生成

例如：

```
好的提示词：
    找出三个使用地热能的国家，并给出每个国家的发电统计数据。
不好的提示词：
    我在阅读关于可再生能源和地热能的内容，听起来很酷。有哪些国家使用它？
```

## 一个例子

还是接着我们的运动员食谱规划任务，我们将提示词改进为：

```
为一名运动员生成满足其饮食限制的一日饮食计划。
```

相比于之前的 `这个人应该吃什么？`，新提示词告诉 Claude：

- 要做什么：生成
- 要生成（创建）什么：一份饮食计划
- 关键约束：一天、一个运动员、满足饮食限制

通过这个改进，评分从 2.32 提升至 3.92。关键点在于，将 Claude 视为一个需要明确指令的助手，而不是一个需要猜测你想要什么的人时，它才能最好地回应。因此，请用直接的动词开头，并具体说明任务。
