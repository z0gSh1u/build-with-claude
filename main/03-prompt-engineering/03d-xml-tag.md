# 03d - 使用 XML 标签

当你的提示词包含大量内容时，为了让 Claude 更容易理解哪些片段属于同一类，可以用 XML 标签优化提示词的结构化程度和清晰度。

## 结构化的重要性

以一个分析 20 页销售记录的提示词为例，如果没有清晰的界限，Claude 难以区分你的指令和实际想要分析的数据。通过将销售记录包裹在 `<sales_records>...</sales_records>` 等 XML 标签中，提供了清晰的分隔符，有助于 Claude 理解提示词的结构。

## 如何使用

你创建的 XML 标签并不强制遵循某种规范（例如 HTML），而应是你的内容的描述性名称，并且适当具体，例如：

- `<sales_records>` 比 `<data>` 更好
- `<athlete_information>` 则清晰地标识了用户的详细信息
- `<my_code>` 和 `<docs>` 分隔了不同类型的内容

XML 标签在以下情况很有用：

- 提示词包含大量上下文或数据
- 提示词混合不同类型的内容（如代码、文档、数据）
- 你想特别清楚地指明内容边界
- 提示词包含多个变量的插值

即使是较短的提示词，XML 标签也可以作为分隔符使你的提示词结构更清晰。

## 实际应用

例如下面这个“为运动员生成食谱”任务的提示词：

```
<athlete_information>
  - 身高：6英尺2英寸
  - 体重：180磅
  - 目标：增肌
  - 饮食限制：素食
</athlete_information>

根据上述运动员信息生成一份饮食计划。
```

这使得提示词非常明确：身高、体重、目标和限制都是与运动员相关的数据，在生成饮食计划时应综合考虑。

随着提示词变得越来越复杂，并包含更多样化的内容，XML 标签的使用价值会越来越高。
