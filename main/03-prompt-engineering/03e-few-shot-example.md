# 03e - 提供示例

在你的提示词中提供示例（输入/输出对样本）是最有效的提示词工程技术之一。

## 如何工作

以情感分析任务为例，我们想让 Claude 对推文进行分类，判断其为正面还是负面：

![img](./03e-few-shot-example.assets/instructor%2Fa46l9irobhg0f5webscixp0bs%2Fpublic%2F1748623645%2F05_-_005_-_Providing_Examples_00.1748623645304.png)

这里的挑战是“反讽”。一条“是的，那是我自《外太空九号》以来看过的最好的电影”这样的推文表面上看起来是正面的，但实际上是讽刺、负面的（《外太空九号》是史上最差的电影之一）。要解决这个问题，可以添加示例来向 Claude 展示如何处理。改进后的提示词包括：

- 一个明确的正面例子：“今晚的游戏太棒了！”→“正面”
- 一个反讽的例子：“太棒了，我今晚正需要航班延误一次！ ”→“负面”
- 解释为什么需要认真处理反讽的情况

示例可以被包裹在像 `<sample_input>` 和 `<ideal_output>` 这样的 XML 标签中。

## 何时使用

提供示例特别适用于：

- 处理特殊情况或边缘场景
- 定义复杂的输出格式（如特定的 JSON 结构）
- 展示你想要的风格或语气
- 演示如何处理模糊的输入

提供示例的方式可以分为：

- 单例 One-Shot：提供单个示例来建立特定模式
- 多例 Multi-Shot：提供多个示例来涵盖不同场景

在提供示例时，不要只提供输入/输出对，还要解释为什么如此输出是好的。这种额外的背景信息有助于 Claude 理解好回答背后的推理，而不仅仅是格式。例如：

```
<ideal_output>
[这里是你示例]
</ideal_output>

这个例子具备很好的结构化，提供了关于食物选择和数量的详细信息，并且与运动员的目标和限制相符。
```

## 最佳实践

### 从提示词评估中获得示例启发

提示词评估的结果可以指导我们选择好的示例。选用最高分输出作为例子，有助于 Claude 理解你定义的“完美”输出是什么样的。

### 其他

- 使用 XML 标签来清晰地表示示例
- 明确说明示例的存在：“这是一个示例输入及其理想响应”
- 包含针对常见的失败案例的示例
- 解释为什么示例输出被认为是理想的
- 保持示例与你的任务相关

提供示例是一种强大的提示词工程技巧。你无需用文字精确描述你想要什么，而是直接展示出来。这使得你的提示词更加可靠，并帮助 Claude 理解仅通过指令难以表达的微妙需求。
