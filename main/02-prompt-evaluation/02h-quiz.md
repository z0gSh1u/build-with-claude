# 02h - 小测验

> 你写了一个提示词并测试了一次。它运行良好，所以你将其部署到生产环境。这种方法的主要风险是什么？
>
> 

- ☑️ 用户将提供意外的输入，导致其崩溃
- 提示词的成本会变得过高
- 提示词会运行得过于缓慢
- 其他开发者不会理解它

> 你需要为你的提示词评估准备测试用例。你有两个选择：手动编写或使用 Claude 生成。你应该使用哪个模型来生成？

- 可用的最昂贵的模型
- 多个模型组合
- ☑️ 一个更快的模型，如 Haiku
- 你正在测试的相同模型

> 你正在运行一个提示词评估流程。你已经使用 Claude 生成了一些响应。下一步是什么？

- 部署到生产环境
- 重写原始提示
- 创建更多测试题目
- ☑️ 将回答通过评分器

> 你想衡量你的提示词在实际中效果如何。你应该关注哪种方法？

- 使用更多例子
- 提示词工程技巧
- 写更长的提示词
- ☑️ 提示词评估方法

> 你正在使用一个模型评分器来评估回答。为了获得比仅仅是中等分数更好的分数，你应该在分数之外要求什么？
>
> 

- 仅要求分数
- 与其他回答的比较
- ☑️ 优势、劣势及理由
- 更详细的解释

> 哪种评分器使用另一个 AI 模型来评估输出的质量？

- ☑️ 模型评分器
- 人工评分器
- 语法评分器
- 代码评分器
