# 04o - 小测验

> 如何判断 Claude 在对话中是否想要再次进行工具调用？

- 检查回复中是否包含单词“工具”

- 检查回复是否比平时更长

- **查看“tool_use”的 stop_reason 字段**

- 计算消息块的数量

> Claude 使用工具时，返回什么样的消息结构？

- **包含文本和工具使用块的多块消息**

- 仅包含文本的简单回复

- JSON 数据，无任何文本

- 仅错误信息

> 让 Claude 使用工具时，JSON Schema 的最主要目的是什么？

- 为用户格式化最终响应

- **告诉 Claude 你的工具函数期望的参数以及如何使用它**

- 存储工具函数调用的结果

- 加密 Claude 与你的服务器之间的数据

> 批处理工具解决了什么问题？

- 使工具运行更快

- 将工具结果翻译成不同的语言

- **减少了在需要多个工具时来回传输的次数**

- 自动修复工具响应中的错误

> 工具使用的流程中，正确的步骤顺序是什么？

- **初始请求 → 工具请求 → 数据检索 → 最终响应**

- 工具请求 → 初始请求 → 最终响应 → 数据检索

- 最终响应 → 初始请求 → 工具请求 → 数据检索

- 数据检索 → 工具请求 → 初始请求 → 最终响应

> Claude 默认只能访问其训练数据中的信息。怎样能让 Claude 获取当前、实时的信息？

- 根据示例进行有根据的猜测

- 更仔细地搜索其训练数据

- 要求用户提供更多细节

- **使用工具获取外部信息**

> Claude 内置的工具（如文本编辑和网络搜索）与自定义工具有何不同？

- **Claude 提供了 Schema，但开发者仍可能需要实现一些功能**

- 需要特殊的 API 密钥

- 只支持特定文件类型

- 使用成本更高
