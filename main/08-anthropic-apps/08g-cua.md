# 08f - Computer Use 简介

Computer Use 是一项强大的功能。它允许 Claude 直接与桌面环境交互，赋予 AI 像人类一样控制计算机的能力。这为自动化、测试和工作流开辟了全新的可能性。

## Computer Use 能做什么

相比于仅仅生成代码或提供建议，Claude 现在可以浏览网站、点击按钮、填写表单，并实时与应用程序交互，这使得它在执行如下任务时非常有用：

- Web 应用的自动化 QA
- 数据录入和表单填写
- 网站访问和信息收集
- UI 测试和验证
- 重复的桌面端任务

## 实际案例：自动化 QA

假设我们构建了一个带有 Auto Complete 功能的 React 组件，用户可以输入 `@` 来 mention 文件或资源。

![img](./08g-cua.assets/1.jpg)

与其手动测试每一种可能的交互，你可以将这项工作委托给 Claude：

```
你的目标是测试一个 React 组件，站点在 https://test-mentioner.vercel.app/

测试流程：
1. 打开一个新的浏览器标签页
2. 访问 https://test-mentioner.vercel.app/
3. 依次执行以下测试用例
4. 完成所有测试后，撰写一份简洁的报告

测试用例：
1. 输入 'Did you read @' 应显示自动完成选项
2. 输入 'Did you read @' 后按回车应添加 '@document.pdf'
3. 添加 '@document.pdf' 后按退格键，应直接在文本下方显示自动完成选项，而不是页面的其他位置
```

## 工作原理

Computer Use 在受控环境中运行，通常是一个带有浏览器的 Docker 容器，与你的系统隔离。你可以通过聊天界面与 Claude 互动、下达指令，Claude 按照指示一步步操作，如截取屏幕截图、点击元素、输入文本、观察结果。例如在 QA 示例中，Claude 会：

- 导航到指定的网址
- 输入测试输入并观察自动完成行为
- 测试回车键功能
- 检查退格键的行为是否会正确定位
- 生成报告，显示哪些测试通过，哪些失败

这种描述式的自动化测试可以节省开发者大量的时间，尤其是在重复的 QA 任务中，或需要快速测试多个场景时。

## 安全性与隔离

Computer Use 在沙盒环境中运行以确保安全；浏览器和桌面环境在 Docker 容器内运行，与你的系统隔离。这意味着 Claude 可以与 Web 应用交互并测试界面，而不会对个人文件或系统安全造成风险。
