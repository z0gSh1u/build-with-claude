# 07c - 项目配置

我们将构建一个基于 CLI 的聊天机器人，以更好地理解 MCP 客户端和服务器如何协同工作。这个动手项目将让你获得 MCP 架构两端的实践经验。

## 我们要构建什么

我们的聊天机器人将允许用户通过命令行界面与一组文档进行交互。该系统由两个主要组件组成：

- 一个处理用户交互的 MCP 客户端
- 一个自定义的 MCP 服务器，用于管理文档操作

![img](./07c-setup.assets/1.jpg)

该服务器将提供两个基本工具：一个用于读取文档内容，另一个用于更新它们。所有文档将存储在内存中以简化操作——无需数据库。

在现实世界的项目中，你通常只实现 MCP 客户端或 MCP 服务器，而不是两者都实现。你可以创建：

- 一个 MCP 服务器，用于向其他开发者展示你的服务
- 一个 MCP 客户端，用于连接现有的 MCP 服务器

我们在这个项目中纯粹是出于教育目的构建这两个组件——以了解它们如何通信和协同工作。

## 项目设置

项目包含一个包含设置说明的全面 README 文件。请按照以下步骤操作：

- 将您的 Anthropic API 密钥添加到 `.env` 文件中
- 使用 UV（推荐）或 pip 安装依赖项
- 运行启动应用程序以验证一切是否正常工作

在终端中导航到你的项目目录。你会看到主要的项目文件，包括 `main.py` 、 `mcp_client.py` 和 `mcp_server.py` 。

要启动应用程序，请使用以下命令之一：

```
# If using UV (recommended)
uv run main.py

# If using standard Python
python main.py
```

当应用程序成功启动时，你会看到一个聊天提示。通过问一个简单的问题，比如“1+1 等于多少？”来测试它——你应该会从 Claude 那里得到一个快速的回复。

基本设置完成后，我们就可以开始实现 MCP 功能，并探索客户端和服务器如何通过模型控制协议进行通信。
