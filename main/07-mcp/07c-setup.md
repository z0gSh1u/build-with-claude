# 07c - 项目配置

从本节开始，我们将构建一个基于 CLI 的聊天机器人，以更好地理解 MCP 客户端和服务器如何协同工作。我们的聊天机器人将用户通过命令行界面与一组文档进行交互。该系统由两个主要组件组成：

- 一个处理用户交互的 MCP 客户端
- 一个自定义的 MCP 服务器，用于管理文档操作

该 MCP 服务器提供两个基本工具：一个用于读取文档内容，另一个用于更新文档内容。为了简化，所有文档储存在内存中。

![img](./07c-setup.assets/1.jpg)

在实际项目中，开发者通常只需实现 MCP 客户端或 MCP 服务器其一：

- 一个 MCP 服务器可以用于向其他开发者暴露你的服务
- 一个 MCP 客户端可以用于连接现有的 MCP 服务器

我们在这个项目中出于教学目的，同时构建这两个组件，以了解它们如何通信和协同。

## 项目设置

主要的项目文件包括 `main.py` 、 `mcp_client.py` 和 `mcp_server.py`。请按以下步骤准备：

- 将 Anthropic API Key 添加到 `.env` 文件
- 使用 uv 安装依赖
- 运行入口程序 main.py，验证一切是否正常工作

当应用成功启动时，你会看到一个聊天提示，可以问一个简单的问题来测试它。项目设置完成后，我们就可以开始实现 MCP 功能，探索客户端和服务器如何通过 MCP 进行通信。
